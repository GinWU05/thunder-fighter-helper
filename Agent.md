我想写一个游戏助手的web页面，用来做一些游戏计算，这个游戏叫雷霆战机，目前打算有两个模块，一个是体力计算，一个是进阶材料计算。
体力计算：
上限一般是100，但是要支持用户自定义设置，现在我现在的体力上限是120，你把默认值调成120.
游戏每5分钟恢复1点体力，游戏新的一天是00：00。
这个模块第一功能就是根据当前时间还有体力（用户自行输入），判断是否会溢出体力，如果在00：00会溢出体力，建议用户消耗xx体力，避免溢出。如果没溢出，就告诉用户下次满体力的时间。
第二个就是功能就是当天上限，游戏除了体力随着时间自然恢复，还有多种途径获得体力：
- 活跃度：奖励100体力
- 小程序签到：30体力
- 好友赠送：30次，每次5点
- 体力购买： x100 / x50
- 其他体力：用户自行填写从其他途径获得的体力
体力购买，其他体力，需要用户自行填写购买次数
其他方式均用多选框让用户自行交互。
根据以上条件，还有自然恢复途径获取的体力（假设一直没溢出），从而计算用户当天（根据目前世界）能获取的体力上限是多少。
文案风格（新）：短句、标题化、结果导向，强调“新游戏日/当日/上限/溢出”；说明用公式口吻，如“总值为：A + B，合计为理论MAX值”；单位和次数放括号里说明。
展示逻辑（新）：自然恢复=（距离新游戏日的分钟数 ÷ 5）取整；预计新游戏日体力=当前体力+自然恢复；溢出体力=max(0,预计新游戏日体力-体力上限)。满体力时间=当前时间+（体力上限-当前体力）×5分钟，跨天显示“（新游戏日）”。展示优先级：已满且会溢出→显示溢出体力提示；未满但会在新游戏日前满→显示满体力时间；不会满/不会溢出→显示“新游戏日前不会溢出”。

进阶材料主要有两类：
一类是3星、4星、5星材料，一类是战神材料。
每一类材料（3星、4星、5星、战神）都有4种材料，可以说是A,B,C,D，4种材料都是1才能凑成1组材料， 凑成1组材料才能被完整使用。
普通难度可以获取第一类的进阶材料，英雄难度可以获取第二类的战神材料，这两类分别每天都有5次机会是可以双倍获取的。双倍获取次数也要设置成自定义；
```
模式	已实装关卡数	规划上限	说明
普通难度	≥112关	144关	高关卡需长期养成或重氪
英雄难度	24关	36关	部分关卡仍在开发中
```
扫荡关卡可以获得材料还有经验材料。
经验材料每一关的材料获取不固定但是进阶材料是固定的。
之前的关卡都是一关能获取1种、1类材料，但是后面有些副本可以一个关卡获取2种材料。

UI 指引（暗色 · 雷霆战机风格）

目标
- 仅暗色主题；色值可贴近游戏 UI，也可保持风格一致。
- 关键词：冷暗底、霓虹蓝/紫高亮、金橙行动点。
- 助手定位：减法、清晰、不抢游戏主界面。

色彩（hex，可微调）
- 背景：#050812
- 遮罩：#050812 @55%（弱）/ @75%（强）
- 面板底：#0A1630
- 面板渐变：#122A4D → #071225（上亮下暗）
- 主文字：#EAF3FF
- 次文字：#9BB3D6
- 霓虹蓝：#2FD2FF
- 霓虹紫：#B26CFF
- 行动金：#FFC84A → #FF9B2F
- 危险红：#FF3B3B
- 成功绿：#3DFFCC
- 禁用灰蓝：#4E6B8F

使用规则
- 文字优先可读性，面板与文本保持清晰对比。
- 霓虹色只用于重点/状态，大面积填充用深色。
- Hover/Active 用亮度微调实现；禁用去发光并降对比。

字体与排版
- 字体：中文优先思源黑体/Source Han Sans/Noto Sans SC，标题/按钮用 Semibold，正文 Regular。
- 数字：使用 tabular-nums 或等宽数字，便于扫读。
- 尺寸参考：大标题 22–26，小标题 16–18，正文 14–16，注释 12–13，数字强调 20–28。

组件样式
- Panel：外框 2px 亮色描边 + 内框 1px 暗描边；允许轻微切角；背景半透明 + 细网格纹理。
- Header Bar：左小图标 + 标题 + 右侧小按钮（关闭/收起）。
- Section：1px 分割线 #1A3A5E，内容每段 ≤2 行，超出折叠。

按钮与控件
- 主按钮：圆角梯形/胶囊梯形；金橙渐变；2px 亮边 + 1px 内暗边；扫光 2–3 秒一次。
- 次按钮：霓虹蓝渐变，发光更弱。
- 文字按钮：蓝色文字 + 下划线/小箭头。
- 角标：红点 10–12；数字角标高度 16–18；仅在“限时/错过损失”使用。

引导与提示
- Spotlight：全屏遮罩 + 圆角镂空；蓝色 2px 描边 + 呼吸光（1.6–2.4s）；必须可关闭。
- Callout：1–2px 细线 + 小锐箭头；可加极淡流动粒子。
- Tooltip：2 行文字 + 1 个按钮，贴近目标但不遮挡。

动效
- 出入场 200–280ms ease-out，避免弹跳。
- 呼吸光、扫光低频使用；`prefers-reduced-motion` 时关闭动效。

拥挤度策略
- 默认折叠，点开再展开。
- 优先贴边浮层，避免居中大弹窗。
- 不同时出现“高亮 + 大弹窗 + 多按钮”，最多两个元素并存。
- 每个浮层只服务一个决策点，文案短句动词开头。

模板
- 快速建议卡：标题 + 结论 +（可选理由）+ 主/次按钮。
- 步骤引导：Step 1/2/3，当前步骤高亮，右上角“跳过”。
- 策略面板：P1/P2/P3 分区 + 收益标签（+战力/+收益/限时/必做）；无法跳转时给替代路径。

交付结构
- Design Tokens、基础组件、模板组件、动效库。
